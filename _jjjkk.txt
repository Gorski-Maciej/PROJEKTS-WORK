🚀 Zaawansowane funkcjonalności (ponad standard)

1. Multi-Cloud Cost Aggregation z normalizacją
   · Agenci pobierają dane kosztowe z AWS (Cost Explorer API), Azure (Billing API), GCP (Cloud Billing API) i normalizują je do jednego formatu.
   · Dla on-premise: agent pobiera metryki z Proxmox/vSphere API lub bezpośrednio przez SSH (htop, iostat).
   · Wszystkie dane trafiają do DuckDB, gdzie są łączone i agregowane.
2. Inteligentne rekomendacje optymalizacyjne
   · Moduł analizuje dane i wykrywa:
     · Nieużywane zasoby (idle instances, unattached volumes, stare snapshoty).
     · Możliwość zakupu Reserved Instances / Savings Plans (z kalkulacją opłacalności).
     · Przeskalowanie zasobów (rightsizing) na podstawie rzeczywistego zużycia.
   · Rekomendacje są zapisywane w PostgreSQL i prezentowane w dashboardzie z szacowanymi oszczędnościami.
3. Predykcja kosztów z sezonowością
   · Prophet trenuje modele na danych historycznych (min. 3 miesiące).
   · Prognozy uwzględniają sezonowość (np. wyższe koszty w Black Friday dla e-commerce).
   · System wysyła alerty, gdy prognozowane wydatki przekraczają budżet.
4. Symulacje "What-If"
   · Użytkownik może zasymulować zmianę architektury (np. "co jeśli przeniosę 5 serwerów z AWS do Azure?") i zobaczyć prognozowane koszty.
   · Backend używa Polars do szybkich kalkulacji na danych historycznych.
5. Automatyczne akcje (Self-Healing FinOps)
   · Gdy system wykryje nieużywany zasób przez X dni, automatycznie wysyła powiadomienie, a po akceptacji użytkownika – wykonuje akcję (np. zatrzymuje instancję przez API chmury).
   · Wszystkie akcje są logowane i wymagają potwierdzenia (audyt).
6. Wielodostępność (Multi-Tenant)
   · System obsługuje wiele firm/zespołów w ramach jednej instancji.
   · Izolacja danych na poziomie bazy (PostgreSQL schemas lub row-level security).
7. Integracja z fakturami (OCR + AI)
   · Użytkownik może wrzucić fakturę za infrastrukturę – system używa Tesseract + LLM (Phi-3) do wyciągnięcia kwot i pozycji, a następnie porównuje je z danymi z API chmurowych (wykrywanie rozbieżności).

2. Agenci – rozbudowana implementacja
agents/base_agent.py
python
from abc import ABC, abstractmethod
from datetime import datetime, timezone
from typing import List, Dict
import asyncio
import logging

logger = logging.getLogger(__name__)

class BaseCloudAgent(ABC):
    """Abstrakcyjny agent dla wszystkich dostawców."""
    
    def __init__(self, config: dict):
        self.config = config
        self.name = config.get('name', self.__class__.__name__)
        self.collect_interval = config.get('collect_interval', 3600)  # 1h
    
    @abstractmethod
    async def collect_costs(self, start_date: datetime, end_date: datetime) -> List[Dict]:
        """Pobiera koszty z danego okresu."""
        pass
    
    @abstractmethod
    async def collect_usage(self) -> Dict:
        """Pobiera aktualne zużycie zasobów."""
        pass
    
    @abstractmethod
    async def list_resources(self) -> List[Dict]:
        """Lista wszystkich zasobów."""
        pass
    
    @abstractmethod
    async def execute_action(self, resource_id: str, action: str, params: dict = None) -> bool:
        """Wykonuje akcję na zasobie (np. stop, resize)."""

        """Główna pętla agenta."""
        logger.info(f"Agent {self.name} started")
        while True:
            try:
                now = datetime.now(timezone.utc)

                )
                usage = await self.collect_usage()

                await self.publish_metrics(costs, usage, resources)

                await asyncio.sleep(self.collect_interval)

                await asyncio.sleep(60)

class AWSCloudAgent(BaseCloudAgent):
    def __init__(self, config: dict):
        super().__init__(config)
        self.ce = boto3.client('ce', region_name=config.get('region', 'us-east-1'))
        self.ec2 = boto3.client('ec2', region_name=config.get('region'))
        self.rds = boto3.client('rds', region_name=config.get('region'))
        self.ebs = boto3.client('ec2', region_name=config.get('region'))
    
    async def collect_costs(self, start_date: datetime, end_date: datetime) -> List[Dict]:
        """Pobiera koszty z AWS Cost Explorer API."""
        response = self.ce.get_cost_and_usage(
            TimePeriod={
                'Start': start_date.strftime('%Y-%m-%d'),
                'End': end_date.strftime('%Y-%m-%d')
            },
            Granularity='DAILY',
            Metrics=['UnblendedCost'],
            GroupBy=[
                {'Type': 'DIMENSION', 'Key': 'SERVICE'},
                {'Type': 'DIMENSION', 'Key': 'REGION'}
            ]
        )
        costs = []
        for result in response['ResultsByTime']:
            for group in result['Groups']:
                costs.append({
                    'date': result['TimePeriod']['Start'],
                    'service': group['Keys'][0],
                    'region': group['Keys'][1] if len(group['Keys']) > 1 else 'unknown',
                    'amount': float(group['Metrics']['UnblendedCost']['Amount']),
                    'currency': response['ResultsByTime'][0]['Total']['UnblendedCost']['Unit']
                })
        return costs
    
    async def collect_usage(self) -> Dict:
        """Pobiera aktualne zużycie EC2, RDS, EBS."""
        # EC2 instances
        ec2_instances = self.ec2.describe_instances()
        running_instances = []
        for reservation in ec2_instances['Reservations']:
            for instance in reservation['Instances']:
                if instance['State']['Name'] == 'running':
                    running_instances.append({
                        'id': instance['InstanceId'],
                        'type': instance['InstanceType'],
                        'state': instance['State']['Name'],
                        'launch_time': instance['LaunchTime'].isoformat()
                    })
        
        # RDS instances
        rds_instances = self.rds.describe_db_instances()
        running_rds = []
        for db in rds_instances['DBInstances']:
            if db['DBInstanceStatus'] == 'available':
                running_rds.append({
                    'id': db['DBInstanceIdentifier'],
                    'class': db['DBInstanceClass'],
                    'engine': db['Engine'],
                    'storage': db['AllocatedStorage']
                })
        
        # EBS volumes
        ebs_volumes = self.ebs.describe_volumes()
        volumes = []
        for vol in ebs_volumes['Volumes']:
            if vol['State'] == 'available':  # nieprzypięte
                volumes.append({
                    'id': vol['VolumeId'],
                    'size': vol['Size'],
                    'type': vol['VolumeType'],
                    'state': vol['State']
                })
        
        return {
            'ec2_instances': running_instances,
            'rds_instances': running_rds,
            'ebs_volumes': volumes,
            'total_ec2': len(running_instances),
            'total_rds': len(running_rds),
            'unattached_volumes': len(volumes)
        }
    
    async def list_resources(self) -> List[Dict]:
        """Pełna lista zasobów z metadanymi."""
        usage = await self.collect_usage()
        resources = []
        
        for inst in usage['ec2_instances']:
            resources.append({
                'resource_id': inst['id'],
                'type': 'EC2',
                'status': inst['state'],
                'details': inst
            })
        
        for db in usage['rds_instances']:
            resources.append({
                'resource_id': db['id'],
                'type': 'RDS',
                'status': 'available',
                'details': db
            })
        
        for vol in usage['ebs_volumes']:
            resources.append({
                'resource_id': vol['id'],
                'type': 'EBS',
                'status': vol['state'],
                'details': vol
            })
        
        return resources
    
    async def execute_action(self, resource_id: str, action: str, params: dict = None) -> bool:
        """Wykonuje akcję na zasobie AWS."""
        try:
            if action == 'stop_ec2':
                self.ec2.stop_instances(InstanceIds=[resource_id])
                logger.info(f"Stopped EC2 instance {resource_id}")
            elif action == 'start_ec2':
                self.ec2.start_instances(InstanceIds=[resource_id])
            elif action == 'delete_ebs':
                self.ebs.delete_volume(VolumeId=resource_id)
                logger.info(f"Deleted EBS volume {resource_id}")
            elif action == 'resize_rds':
                new_class = params.get('new_class', 'db.t3.micro')
                self.rds.modify_db_instance(
                    DBInstanceIdentifier=resource_id,
                    DBInstanceClass=new_class,
                    ApplyImmediately=True
                )
            return True
        except Exception as e:
            logger.error(f"Action {action} failed for {resource_id}: {e}")
            return False

class ActionExecutor:
    def __init__(self, db_pool, redis_client):
        self.db = db_pool
        self.redis = redis_client
        self.agents = {
            'aws': AWSCloudAgent({}),
            'azure': AzureCloudAgent({}),
            'gcp': GCPCloudAgent({})
        }
    
    async def execute_recommendation(self, rec_id: str, tenant_id: str, 
                                    approved_by: str) -> bool:
        """Wykonuje zatwierdzoną rekomendację."""
        # Pobierz rekomendację
        async with self.db.acquire() as conn:
            rec = await conn.fetchrow(
                "SELECT * FROM recommendations WHERE id = $1", rec_id
            )
            if not rec or rec['status'] != 'approved':
                return False
            
            # Określ agenta na podstawie resource_type
            agent = self._get_agent_for_resource(rec['resource_type'])
            
            # Wykonaj akcję
            success = await agent.execute_action(
                rec['resource_id'],
                rec['action'],
                rec.get('params', {})
            )
            
            if success:
                # Zaktualizuj status
                await conn.execute("""
                    UPDATE recommendations SET status = 'executed', executed_at = $1
                    WHERE id = $2
                """, datetime.now(timezone.utc), rec_id)
                
                # Zaloguj audyt
                await self.audit_log(tenant_id, approved_by, rec['resource_id'], 
                                    rec['action'], rec['estimated_monthly_savings'])
                
                # Wyślij powiadomienie
                await self.send_notification(tenant_id, f"Executed: {rec['title']}")
            
            return success
    
    def _get_agent_for_resource(self, resource_type: str):
        """Zwraca odpowiedniego agenta na podstawie typu zasobu."""
        if resource_type in ('EC2', 'RDS', 'EBS'):
            return self.agents['aws']
        elif resource_type.startswith('Microsoft.'):
            return self.agents['azure']
        elif resource_type.startswith('Google.'):
            return self.agents['gcp']
        return self.agents['aws']  # default
    
    async def schedule_auto_actions(self, tenant_id: str):
        """Automatycznie wykonuje akcje dla zasobów idle > 30 dni."""
        recommendations = await RecommendationEngine(self.db, self.redis).detect_idle_resources(tenant_id)
        for rec in recommendations:
            # Automatycznie zatrzymaj po 30 dniach idle

🛡️ Warstwa Niezawodności (Reliability)
To absolutny fundament. Bez tego system nie jest produkcyjny.

Circuit Breaker dla Agentów

Problem: Jeśli AWS/GCP/Azure API jest niedostępne, agent będzie bez końca ponawiał próby, co marnuje zasoby i opóźnia inne zadania.

Rozwiązanie: Dodaj bibliotekę pybreaker do agentów. Po 3 nieudanych próbach w ciągu minuty obwód się otwiera i agent na 5 minut przestaje próbować, przechodząc w tryb awaryjny (np. pobiera tylko dane z cache).

Implementacja: Kilka linijek w base_agent.py.

Dead Letter Queue (DLQ)

Problem: Jeśli wiadomość w RabbitMQ jest błędna lub nie może zostać przetworzona (np. błędny format JSON z agenta), worker będzie ją bez końca przetwarzał, blokując kolejkę.

Rozwiązanie: Skonfiguruj RabbitMQ z DLQ. Każda kolejka powinna mieć ustawiony x-dead-letter-exchange. Worker, po 3 nieudanych próbach, odrzuca wiadomość (basic.reject), a ona trafia do dedykowanej kolejki cost.ingest.dlq. Dodaj endpoint API do przeglądania i ponownego przetwarzania DLQ.

Konfiguracja: Głównie w definitions.json dla RabbitMQ i w konfiguracji brokera w aplikacji.

Retry Policy z Backoffem

Problem: Tymczasowy błąd sieciowy przy zapisie do bazy danych powoduje niepowodzenie całego zadania.

Rozwiązanie: Użyj biblioteki tenacity (Python). Wszędzie tam, gdzie łączysz się z zewnętrznymi systemami (bazy danych, API chmurowe, RabbitMQ), dodaj dekorator @retry z wykładniczym backoffem i maksymalną liczbą prób.

Health Check i Readiness Probe

Problem: Kubernetes/Docker nie wie, czy aplikacja jest gotowa na przyjęcie ruchu, ani czy wciąż żyje.

Rozwiązanie: Dodaj do FastAPI endpointy /healthz (czy proces działa) i /readyz (czy wszystkie zależności – bazy danych, kolejki – są osiągalne). Skonfiguruj je w docker-compose i Helm chartach.

Graceful Shutdown

Problem: Gdy kontener jest zatrzymywany, musi dokończyć przetwarzanie bieżących zadań, aby nie stracić danych.

Rozwiązanie: W workerze (queue_worker.py) nasłuchuj na sygnał SIGTERM. Po jego otrzymaniu worker przestaje pobierać nowe wiadomości z RabbitMQ, czeka, aż bieżące zadania się zakończą, zamyka połączenia i dopiero wtedy kończy pracę.

Gdy rekomendacja osiągnie status "approved", automatycznie twórz zgłoszenie w Jira/ServiceNow

Linkuj zgłoszenie z rekomendacją w CloudBudget

Aktualizuj status zgłoszenia, gdy rekomendacja zostanie wykonana

Użyj webhooków do dwukierunkowej synchronizacji

Implementacja: Nowy moduł api/services/ticketing_service.py z adapterami dla Jira i ServiceNow.

7. Moduł budżetowania i rozliczeń (Chargeback/Showback)
Problem: Firmy potrzebują rozliczać koszty chmury na poszczególne zespoły.
Rozwiązanie:

Definiowanie budżetów: Na poziomie tagu (np. Team=DevOps), konta AWS, subskrypcji Azure

Śledzenie wydatków: Real-time vs budżet (wykorzystanie %)

Alerty: Gdy wydatki osiągną 80% / 100% / 120% budżetu

Raporty okresowe: Automatycznie generowane PDF z rozliczeniem per zespół

Integracja z tagami: Użyj tagów do mapowania kosztów na centra kosztowe

Implementacja: Nowy moduł api/services/budget_service.py i rozszerzenie bazy danych.

8. Analiza kosztów Kubernetes (pod-level)
Problem: Koszty klastra K8s są trudne do przypisania do konkretnych aplikacji.
Rozwiązanie: Agent kubernetes_agent.py rozszerzony o:

Integrację z kube-cost (OpenCost) lub metrics-server

Pobieranie metryk CPU/RAM per namespace, deployment, pod

Przeliczanie na koszt na podstawie cennika instancji worker nodes

Rekomendacje: resize requestów/limitów, zmiana typu instancji, scale-down nieużywanych namespace'ów

Implementacja: Rozszerz agents/kubernetes_agent.py.

9. Import/Eksport danych z/do S3 i formatów Parquet/CSV
Problem: Użytkownicy chcą analizować dane w zewnętrznych narzędziach (Excel, PowerBI).
Rozwiązanie:

Eksport: Endpointy API do eksportu danych kosztowych do CSV, Parquet, JSON. Możliwość zapisu bezpośrednio na S3.

Import: Możliwość wgrania pliku CSV z danymi kosztowymi (np. z legacy systemów). Walidacja i mapowanie kolumn.

Harmonogram: Automatyczny eksport co miesiąc na S3.

Implementacja: Nowy router api/routers/exports.py i użycie polars do konwersji.

10. Analiza porównawcza faktur vs. rzeczywiste koszty (Reconciliation)
Problem: Faktury od dostawców chmury nie zawsze zgadzają się z własnymi kalkulacjami.
Rozwiązanie: Rozszerz ml/invoice_ocr.py o:

Porównanie pozycji z faktury z danymi z API chmury dla tego samego okresu

Wykrywanie rozbieżności > 1%

Raport reconciliation (PDF) z zaznaczonymi różnicami

Automatyczne zgłoszenie do supportu chmury, gdy rozbieżność jest znacząca
